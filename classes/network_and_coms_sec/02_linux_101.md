## `man` and `file`

There are a huge number of Linux commands and they each have many options. The most important habit to build is checking the manual before guessing:

```bash
man ls
man unzip
```

The `file` command is especially useful in security/forensics work — it identifies a file's true type by **reading the file header**, not just the extension. A file renamed from `.exe` to `.txt` will still show its real type:

```bash
file access.zip
```


## `less` and `more` — Paginating Large Files

When files are too big to dump to the terminal, you need a pager.

**`more`** is the older, simpler tool — it shows one page at a time and only goes forward:
```bash
more access.log
```

**`less`** is better in almost every way — you can scroll forward and backward, search, jump to lines, and follow live updates:
```bash
less access.log
less -N access.log              # show line numbers
less +5000 access.log           # start at line 5000
less +F /var/log/messages       # follow file as it grows (like tail -f)
                                # or press Shift+F once inside less
less +/POST access.log          # open and highlight a pattern
```

> **Exercise from class:** Figure out how to display line numbers and start at line 5000 using the man page.


## Pipe `|` and Stream Redirection `>` `>>` `<`

This is one of the most important concepts in Linux — chaining commands together and controlling where data goes.

### Pipe `|`
Passes the output of one command directly as input to the next:
```bash
ls -la /etc/ | less
```

### Three Streams
Every process in Linux has three standard streams:

| Stream | Number | Default symbol |
|--------|--------|----------------|
| stdin  | 0 | `<` |
| stdout | 1 | `>` |
| stderr | 2 | `2>` |

### Redirect Examples
```bash
ls -la /etc/ > ls_file.txt          # save stdout to file (overwrites)
ls -la /etc/ 1> ls_file.txt         # same — 1 is default for >
ls -la /etc/ >> ls_file.txt         # append instead of overwrite
rm missing_file 2> rm_error.log     # redirect errors only
rm missing_file 2>> rm_error.log    # append errors
./program.py < program_input.txt    # feed a file as stdin
./program.py 0< program_input.txt   # same — 0 is default for <
```

> **Exercise from class:** Create `program.py` in Python that asks for the user's name and age, then prints them. Then call it so stdout goes to one file and stderr to another.


## `echo`

`echo` prints text to the terminal, but it's also very useful for feeding input into other programs via pipes:

```bash
echo "Hi how are you?"
echo -e "Dany\n40\n" | ./program.py          # pipe as stdin to program
echo -e "Dany\n40\n" | ./program.py > out.txt  # pipe + save output
```

**Flags to know:**
- `-e` — enables escape sequences like `\n` (newline), `\t` (tab)
- `-E` — disables escape sequences (this is the default)

> **Exercise from class:** Figure out what `-e` and `-E` do. Then call `program.py` so stdout saves to one file and errors to another.


## `head` — View the Beginning of a File

Shows the first lines of a file. Default is 10 lines:

```bash
head access.log                     # first 10 lines
head -n 35 access.log               # first 35 lines
head -n 35 access.log | less -N     # pipe to less with line numbers
head -c 50 access.log               # first 50 bytes/characters
```


## `tail` — View the End of a File

Mirror of `head` — shows the last lines. Same flags apply:

```bash
tail access.log                     # last 10 lines
tail -n 35 access.log | less -N
tail -f /var/log/messages           # follow file live as it grows ← very useful for monitoring logs
```


## `cat` — Display and Concatenate Files

`cat` dumps the full content of a file to stdout. Also useful for creating files and merging them:

```bash
cat /var/log/messages               # display file
cat > my_file.txt                   # create/overwrite file (type content, Ctrl+D to finish)
cat file1.txt file2.txt file3.txt > new_file.txt   # merge multiple files into one
```


## `grep` — Search for Patterns

`grep` is essential for finding things in large datasets. It searches line by line and prints matching lines:

```bash
grep POST access.log                # all lines containing "POST"
grep POST access.log | head         # first 10 POST matches
sudo ifconfig | grep eth0           # filter output of another command
sudo ifconfig | grep -A 2 eth0      # show 2 lines AFTER each match
grep -B 2 eth0                      # show 2 lines BEFORE each match
grep -v POST access.log | head      # INVERT — lines NOT containing "POST"
grep -n POST access.log             # show line numbers alongside matches
grep -vn POST access.log | head     # combine flags
grep -f patterns.txt access.log     # load multiple patterns from a file
head -n 500 access.log | grep -f patterns.txt
```


## `wc` — Count Lines, Words, Bytes

Useful for getting totals out of any pipeline:

```bash
wc access.log           # prints: lines  words  bytes
wc -l access.log        # lines only — much faster on large files
```

> **Exercise from class:** Combine `ls`, `grep`, and `wc` to count how many files or directories in `/etc/` contain "net" in their name.
> 
> Answer: `ls /etc/ | grep net | wc -l`


## `cut` and `awk` — Extract Columns

### `cut` — Simple, fast column extraction
Works like Python's `split()`. Define a delimiter with `-d` and pick which field(s) with `-f`:

```bash
cut -d ' ' -f1 access.log | head           # column 1 = IP address
cut -d ' ' -f1,4-5 access.log | head       # columns 1, 4–5 (IP + date/time)
```

### `awk` — More powerful, can write mini scripts
Slower to write but handles complex cases. Columns are `$1`, `$2`, etc.:

```bash
awk '{print $1" "$4" "$5}' access.log | head   # same result as cut example above
```

Use `cut` for simple jobs; use `awk` when you need logic or calculated output.


## `sort` — Sort Data

```bash
ls -la | sort               # alphabetical sort
ls -la | sort -k5           # sort by column 5 (file size in ls -la)
ls -la | sort -nk5          # numeric sort by column 5
ls -la | sort -rnk5         # reverse numeric = descending order
head access.log | sort -rnk10   # sort first 10 lines by column 10 desc (response bytes)
```

**Flags:** `-k` = column, `-n` = numeric, `-r` = reverse/descending

## `uniq` — Remove or Count Duplicates

`uniq` collapses consecutive identical lines. **Critical gotcha: it only works on adjacent lines**, so you almost always need to `sort` first.

```bash
echo -e "aaa\naaa\naaa\nbb\nbb\nc" > testfile.txt
uniq testfile.txt               # removes consecutive duplicates
uniq -c testfile.txt            # count how many times each line appears
```

**The trick with non-adjacent duplicates:**
```bash
echo -e "aaa" >> testfile.txt   # append "aaa" to the END of the file
uniq -c testfile.txt            # now shows 2 separate "aaa" groups!
```
Because the new `aaa` isn't adjacent to the original block, `uniq` sees it as a different entry. Solution: always sort first:
```bash
sort testfile.txt | uniq -c
```


## `split` — Break Large Files into Smaller Pieces

Useful when a file is too big to process at once, or to parallelise work:

```bash
split -l 100000 access.log      # split into chunks of 100,000 lines (default is 1,000)
split -b 50m access.log         # split by size (50MB chunks)
```

Output files are named `xaa`, `xab`, `xac`, etc. — you'll see many of these after running `ls`.


## `tee` — Write to Screen AND a File Simultaneously

Normally piping to a file means you don't see the output. `tee` lets you do both:

```bash
ls -l | wc -l | tee test.txt   # prints line count to terminal AND saves to test.txt
```

Think of it like a T-junction in a pipe — data flows both directions.


## Background Jobs — `jobs`, `bg`, `fg`

You can run processes in the background so they don't block your terminal:

```bash
gedit test.py &         # launch immediately in background (add & at end)
# Ctrl + Z              # suspend a running foreground process
jobs                    # list all background/suspended jobs with their numbers
fg %1                   # bring job 1 back to foreground
bg %1                   # resume suspended job 1 in the background
```
